Bunny Door System

คู่มือติดตั้งระบบบน Raspberry Pi แบบละเอียดทุกขั้นตอน

สิ่งที่ต้องเตรียม:
  • Raspberry Pi 4 Model B (แนะนำ RAM 4GB ขึ้นไป)
  • MicroSD 32GB ขึ้นไป (แนะนำ 64GB)
  • กล้อง USB 2 ตัว
  • สาย Ethernet หรือ WiFi
  • Power Supply 5V 3A (USB-C)
  • จอ HDMI + คีย์บอร์ด (สำหรับตั้งค่าครั้งแรก) หรือเข้า SSH
สารบัญ
  1. เตรียม Raspberry Pi OS
  2. อัปเดตระบบ
  3. ติดตั้ง Dependencies ระดับ OS
  4. ติดตั้ง MySQL Server
  5. สร้าง Python Virtual Environment
  6. ติดตั้ง dlib + face_recognition (ขั้นตอนสำคัญ)
  7. ติดตั้ง Python Packages ที่เหลือ
  8. โคลนโปรเจค + ตั้งค่า
  9. นำเข้าฐานข้อมูล
  10. เตรียมรูปใบหน้า
  11. ทดสอบกล้อง USB
  12. รันระบบ
  13. ตั้งค่าให้รันอัตโนมัติ (systemd)
  14. เชื่อมต่อ ESP32
  15. เชื่อมต่อ Web Dashboard (PC)
  16. แก้ปัญหาที่พบบ่อย

1 เตรียม Raspberry Pi OS

~15 นาที

1.1 ดาวน์โหลด Raspberry Pi Imager

ดาวน์โหลดจาก https://www.raspberrypi.com/software/ แล้วติดตั้งบน PC

1.2 เขียน OS ลง MicroSD

  1. เปิด Raspberry Pi Imager
  2. เลือก Raspberry Pi OS (64-bit) - ต้องเป็น 64-bit เท่านั้น
  3. เลือก MicroSD Card
  4. กดปุ่มเกียร์ (Settings) เพื่อตั้งค่า:
    • ตั้งชื่อเครื่อง: bunnydoor
    • เปิด SSH
    • ตั้ง Username/Password: pi / รหัสผ่านที่ต้องการ
    • ตั้งค่า WiFi (ถ้าไม่ได้ต่อ Ethernet)
    • Timezone: Asia/Bangkok
  5. กด Write
ต้องใช้ 64-bit OS เท่านั้น เพราะ dlib/face_recognition ทำงานได้ดีกว่ามากบน 64-bit และ library บางตัวไม่รองรับ 32-bit แล้ว

1.3 บูตเครื่อง

ใส่ MicroSD เข้า Raspberry Pi แล้วเปิดเครื่อง รอสักครู่แล้ว SSH เข้า:

ssh pi@bunnydoor.local
# หรือถ้ารู้ IP:
ssh pi@192.168.1.xxx

6 ติดตั้ง dlib + face_recognition

สำคัญมาก ~45-90 นาที
ขั้นตอนนี้ใช้เวลานานที่สุดเพราะต้อง compile dlib จาก source บน Raspberry Pi ห้ามปิดเครื่องระหว่าง compile!

6.1 เพิ่ม Swap (สำคัญมาก!)

dlib ต้องใช้ RAM มากตอน compile ต้องเพิ่ม swap ก่อน:

# เช็ค swap ปัจจุบัน
free -h

# เพิ่ม swap เป็น 2GB
sudo dphys-swapfile swapoff
sudo nano /etc/dphys-swapfile

# หาบรรทัด CONF_SWAPSIZE แล้วแก้เป็น:
CONF_SWAPSIZE=2048

# บันทึก แล้วเปิดใช้งาน swap ใหม่
sudo dphys-swapfile setup
sudo dphys-swapfile swapon

# ตรวจสอบ
free -h
# ควรเห็น Swap: 2.0Gi

6.2 ติดตั้ง dlib

cd ~/bunny-door
source venv/bin/activate

# วิธีที่ 1: ติดตั้งจาก pip (อาจใช้เวลา 30-60 นาที)
pip install dlib

# ถ้าวิธีที่ 1 ล้มเหลว ใช้วิธีที่ 2: compile จาก source
git clone https://github.com/davisking/dlib.git
cd dlib
mkdir build && cd build
cmake ..
cmake --build . --config Release
cd ..
python setup.py install
cd ~/bunny-door
ระหว่าง compile จะเห็น output มากมาย นี่เป็นเรื่องปกติ รอจนเสร็จ ถ้า CPU ร้อนมากเกินไปจะ throttle แต่ไม่เป็นอันตราย

6.3 ติดตั้ง face_recognition

pip install face_recognition

6.4 ทดสอบว่าติดตั้งสำเร็จ

python3 -c "import dlib; print('dlib version:', dlib.__version__)"
# ควรเห็น: dlib version: 19.24.x

python3 -c "import face_recognition; print('face_recognition OK')"
# ควรเห็น: face_recognition OK

6.5 คืนค่า Swap (หลัง compile เสร็จ)

# ลด swap กลับเป็น 100MB เพื่อยืดอายุ SD Card
sudo dphys-swapfile swapoff
sudo nano /etc/dphys-swapfile

# แก้กลับเป็น:
CONF_SWAPSIZE=512

sudo dphys-swapfile setup
sudo dphys-swapfile swapon
ถ้าผ่านขั้นตอนนี้แล้ว ส่วนที่ยากที่สุดเสร็จแล้ว!

10 เตรียมรูปใบหน้า

สำคัญ ~10 นาที

ระบบจะจดจำใบหน้าจากไฟล์รูปในโฟลเดอร์ python/images/

กฎการตั้งชื่อไฟล์

  • ชื่อไฟล์ = ชื่อของคนนั้น (ไม่มีนามสกุลไฟล์ในชื่อ)
  • ไฟล์ต้องเป็น .jpg หรือ .png
  • รูปต้องเห็นใบหน้าชัดเจน 1 หน้าต่อ 1 รูป
  • ชื่อไฟล์ต้องตรงกับ face_image ในตาราง employees

ตัวอย่าง:

~/bunny-door/src/python/images/
    kulthida.jpg       # ตรงกับ face_image = 'kulthida.jpg'
    nattanun.jpg       # ตรงกับ face_image = 'nattanun.jpg'
    nattapol.jpg
    sarayut.jpg

วิธีคัดลอกรูปจาก PC ไปยัง Raspberry Pi

# บน PC (Windows) ใช้ scp:
scp C:\path\to\photos\*.jpg pi@bunnydoor.local:~/bunny-door/src/python/images/

# หรือใช้ WinSCP (GUI) ลากไฟล์ไปวาง

ถ่ายรูปจากกล้อง USB โดยตรง

cd ~/bunny-door
source venv/bin/activate

# สคริปต์ถ่ายรูป (สร้างไฟล์ capture_face.py)
python3 -c "
import cv2
name = input('ชื่อ (ภาษาอังกฤษ): ')
cap = cv2.VideoCapture(0)
print('กด SPACE เพื่อถ่ายรูป, กด Q เพื่อยกเลิก')
while True:
    ret, frame = cap.read()
    if not ret: break
    cv2.imshow('Capture Face', frame)
    key = cv2.waitKey(1) & 0xFF
    if key == ord(' '):
        path = f'src/python/images/{name}.jpg'
        cv2.imwrite(path, frame)
        print(f'บันทึกที่: {path}')
        break
    elif key == ord('q'):
        break
cap.release()
cv2.destroyAllWindows()
"
สคริปต์ถ่ายรูปข้างบนต้องมีจอแสดงผลต่อกับ Pi ถ้าใช้ SSH อย่างเดียว ให้ถ่ายรูปจาก PC แล้ว scp ไปแทน

11 ทดสอบกล้อง USB

~5 นาที 
# ดูกล้องที่เชื่อมต่ออยู่
ls -la /dev/video*
# ควรเห็น /dev/video0 และ /dev/video2 (หรือ video1)

# ดูรายละเอียดกล้อง
v4l2-ctl --list-devices

# USB Camera 1 (usb-0000:01:00.0-1.1):
#     /dev/video0
#     /dev/video1
#
# USB Camera 2 (usb-0000:01:00.0-1.2):
#     /dev/video2
#     /dev/video3
กล้อง USB แต่ละตัวจะสร้าง 2 device nodes (video0+video1, video2+video3) ใช้ตัวเลขคู่ (video0, video2) สำหรับ capture

ทดสอบจับภาพ

cd ~/bunny-door && source venv/bin/activate

python3 -c "
import cv2
# ทดสอบกล้องตัวที่ 1
cap = cv2.VideoCapture(0)
ret, frame = cap.read()
if ret:
    print(f'Camera 0 OK: {frame.shape}')
    cv2.imwrite('/tmp/test_cam0.jpg', frame)
else:
    print('Camera 0 FAILED')
cap.release()

# ทดสอบกล้องตัวที่ 2
cap = cv2.VideoCapture(2)
ret, frame = cap.read()
if ret:
    print(f'Camera 2 OK: {frame.shape}')
    cv2.imwrite('/tmp/test_cam2.jpg', frame)
else:
    print('Camera 2 FAILED - ลอง ID อื่น')
cap.release()
"

แก้ config ถ้า Camera ID ไม่ตรง

ถ้ากล้องตัวที่ 2 ไม่ใช่ ID 1 แต่เป็น ID 2 (หรืออื่น) ให้แก้ไฟล์ .env:

nano ~/bunny-door/src/python/.env

# เพิ่มบรรทัด:
CAMERA_OUTSIDE_ID=0
CAMERA_INSIDE_ID=2

แล้วแก้ config.py ให้อ่านค่าจาก .env:

# ในไฟล์ config.py แก้:
CAMERA_OUTSIDE_ID = int(os.environ.get("CAMERA_OUTSIDE_ID", "0"))
CAMERA_INSIDE_ID = int(os.environ.get("CAMERA_INSIDE_ID", "1"))

13 ตั้งค่าให้รันอัตโนมัติ (systemd)

~5 นาที

สร้าง service file เพื่อให้ระบบเปิดทำงานอัตโนมัติเมื่อ boot:

sudo nano /etc/systemd/system/bunny-door.service

ใส่เนื้อหา:

[Unit]
Description=Bunny Door Face Recognition Server
After=network.target mariadb.service
Wants=mariadb.service

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/bunny-door/src/python
Environment=PATH=/home/pi/bunny-door/venv/bin:/usr/bin:/bin
ExecStart=/home/pi/bunny-door/venv/bin/python face_server.py
Restart=on-failure
RestartSec=10

# Logging
StandardOutput=journal
StandardError=journal

# Security
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target

เปิดใช้งาน

# โหลด service ใหม่
sudo systemctl daemon-reload

# เปิดให้ทำงานตอน boot
sudo systemctl enable bunny-door

# เริ่มทำงานทันที
sudo systemctl start bunny-door

# ตรวจสอบสถานะ
sudo systemctl status bunny-door

# ดู logs
journalctl -u bunny-door -f
# กด Ctrl+C เพื่อหยุดดู

คำสั่งที่ใช้บ่อย

คำสั่งความหมาย
sudo systemctl start bunny-doorเริ่มระบบ
sudo systemctl stop bunny-doorหยุดระบบ
sudo systemctl restart bunny-doorรีสตาร์ท
sudo systemctl status bunny-doorดูสถานะ
journalctl -u bunny-door -fดู logs แบบ real-time
journalctl -u bunny-door --since todayดู logs วันนี้

14.2 หา IP ของ Raspberry Pi

14.3 อัปโหลดโค้ดไปยัง ESP32

  1. เปิด Arduino IDE บน PC
  2. ติดตั้ง Board: ESP32 (จาก Board Manager)
  3. เปิดไฟล์ esp32/door_controller/door_controller.ino
  4. เลือก Board: ESP32 Dev Module
  5. เลือก Port: COM port ที่ ESP32 ต่ออยู่
  6. กด Upload

14.4 แก้ config.py บน Raspberry Pi

nano ~/bunny-door/src/python/.env

# เพิ่ม IP ของ ESP32:

# ดู IP ของ ESP32 ได้จาก Serial Monitor ของ Arduino IDE

  • Dashboard แสดงข้อมูลสถิติ
  • ภาพจากกล้องทั้ง 2 ตัว (MJPEG stream)
  • สถานะประตู (ล็อก/ปลดล็อก)
  • ข้อมูลพนักงาน 4 คน

? แก้ปัญหาที่พบบ่อย

กล้องไม่ทำงาน

# ตรวจสอบว่ากล้องเชื่อมต่ออยู่
lsusb
# ควรเห็นกล้อง 2 ตัว

# ตรวจสอบ /dev/video
ls /dev/video*

# ถ้าไม่เห็น video device:
sudo modprobe uvcvideo

# ตรวจสอบ permissions
groups pi
# ต้องมี "video" ในรายการ ถ้าไม่มี:
sudo usermod -aG video pi
sudo reboot

dlib compile ล้มเหลว

# ตรวจสอบว่ามี swap พอ
free -h
# ต้องมี Swap อย่างน้อย 2GB

# ตรวจสอบ cmake version
cmake --version
# ต้อง >= 3.5

# ถ้า error เรื่อง boost:
sudo apt install -y libboost-python-dev

# ถ้า error เรื่อง X11:
sudo apt install -y libx11-dev

face_recognition ช้ามาก

บน Raspberry Pi 4 ใช้ model "hog" จะได้ ~1-3 FPS ถ้าใช้ "cnn" จะช้ามาก (~0.1 FPS)

ตรวจสอบการตั้งค่าใน .env หรือ config.py:

FACE_MODEL = "hog"              # ใช้ hog เท่านั้นบน Pi
FRAME_RESIZING = 0.25           # ลดขนาดภาพเป็น 25%
PROCESS_EVERY_X_FRAMES = 5     # ประมวลผลทุก 5 เฟรม

MySQL เชื่อมต่อไม่ได้

# ตรวจสอบว่า MariaDB ทำงานอยู่
sudo systemctl status mariadb

# ทดสอบเชื่อมต่อ
mysql -u bunny -p -e "SELECT 1"

# ถ้า PC เชื่อมต่อไม่ได้ ตรวจสอบ:
# 1. bind-address ต้องเป็น 0.0.0.0
grep bind-address /etc/mysql/mariadb.conf.d/50-server.cnf

# 2. Firewall
sudo ufw status
# ถ้าเปิดอยู่:
sudo ufw allow 3306
sudo ufw allow 5000

ESP32 เชื่อมต่อไม่ได้

# ตรวจสอบว่าอยู่วง WiFi เดียวกัน
ping IP-ของ-ESP32

# ตรวจสอบ port 5000
ss -tlnp | grep 5000
# ต้องเห็น: LISTEN 0.0.0.0:5000

# Firewall
sudo ufw allow 5000

ระบบ crash / ค้าง

# ดู logs
journalctl -u bunny-door --since "10 min ago"

# ดู CPU/RAM
htop

# ดูอุณหภูมิ
vcgencmd measure_temp
# ถ้าเกิน 80°C ต้องมีพัดลมระบายความร้อน

# Restart service
sudo systemctl restart bunny-door
