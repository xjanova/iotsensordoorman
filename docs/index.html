1. ภาพรวมโครงงาน

Bunny Door System

ระบบตรวจสอบบุคคลเข้า-ออกห้องสโตร์ด้วยการประมวลผลภาพ

Authorized Access System for Store Room

ระบบนี้เป็นโครงงานของสาขาเทคโนโลยีไฟฟ้าอุตสาหกรรม (ต่อเนื่อง) คณะเทคโนโลยีอุตสาหกรรม มหาวิทยาลัยราชภัฏวไลยอลงกรณ์ ในพระบรมราชูปถัมภ์ จังหวัดปทุมธานี

ออกแบบมาเพื่อควบคุมการเข้า-ออกห้องเก็บของ (สโตร์) โดยใช้เทคโนโลยี Face Recognition ร่วมกับ IoT Sensors, ESP32 และ Raspberry Pi ทำงานผ่านเครือข่าย HTTP มี Web Dashboard สำหรับจัดการข้อมูล

Face Recognition

ระบุตัวตนด้วย HOG/CNN + Face Embedding 128-D Vector

Dual Sensors

PIR 2 ตัว (นอก+ใน) จับการเคลื่อนไหวทั้ง 2 ด้าน

Dual Cameras

กล้อง USB 2 ตัว ถ่ายทั้งขาเข้าและขาออก

2. วัตถุประสงค์

1

ออกแบบระบบบันทึกภาพและเวลาของผู้ที่เข้า-ออกห้องเก็บของ (สโตร์)

2

สร้างระบบระบุและตรวจสอบใบหน้าบุคคลสำหรับควบคุมสิทธิ์การเข้า-ออก

3

พัฒนาระบบควบคุมการเข้า-ออกด้วยการประมวลผลภาพ + IoT + HTTP Network

4

ควบคุมอุปกรณ์หน้างาน (Relay, กลอนแม่เหล็ก) ผ่านคำสั่งจากระบบส่วนกลาง

5

เพิ่มเติม: ตรวจจับการเข้า-ออกที่ผิดปกติ (Tailgating) ด้วยเซ็นเซอร์ + กล้อง 2 ด้าน

ลำดับการทำงาน

1

PIR Sensor ตรวจจับการเคลื่อนไหว → ESP32 ส่ง HTTP POST ไป Raspberry Pi

2

Camera จับภาพ → Face Recognition ตรวจสอบใบหน้า

3

ตรวจสอบสิทธิ์ จากฐานข้อมูล MySQL → ถ้า Authorized ส่งคำสั่ง unlock

4

Raspberry Pi → HTTP POST /api/door/unlock → ESP32

5

ESP32 → GPIO25 HIGH → Relay ON → กลอนแม่เหล็กปลดล็อก (7 วินาที)

6

ครบเวลา → Relay OFF → ประตูล็อกอัตโนมัติ + บันทึกลง DB

4. อุปกรณ์ฮาร์ดแวร์

อุปกรณ์ จำนวน หน้าที่ หมายเหตุ
Raspberry Pi 4 1 Main Controller + Face Server RAM 4GB ขึ้นไป
ESP32 DevKit 1 ควบคุม Sensor + Relay + HTTP WiFi + Bluetooth
USB Camera / Webcam 2 ถ่ายภาพด้านนอก + ด้านใน ต่อ USB กับ Raspberry Pi
PIR Sensor (HC-SR501) 2 ตรวจจับการเคลื่อนไหว นอก + ใน ต่อกับ ESP32
Relay Module 5V 1 ตัด-ต่อไฟกลอนแม่เหล็ก 1 Channel
กลอนแม่เหล็กไฟฟ้า 12V 1 ล็อก/ปลดล็อกประตู Fail-Safe (ไฟดับ = ปลดล็อก)
Power Supply 12V 2A 1 จ่ายไฟให้กลอนแม่เหล็ก DC Adapter
Buzzer 5V 1 แจ้งเตือนเสียง Active Buzzer
LED สีเขียว + สีแดง อย่างละ 1 แสดงสถานะ อนุญาต/ไม่อนุญาต + Resistor 220Ω
ปุ่มกด (Emergency) 1 ปุ่มฉุกเฉิน (เปิดประตูจากด้านใน) Pull-up

5.2 การต่อ PIR Sensor (HC-SR501)

PIR 1 - ด้านนอกประตู

  HC-SR501          ESP32
  ┌─────────┐
  │  VCC ───│─────── 3.3V
  │  OUT ───│─────── GPIO 27
  │  GND ───│─────── GND
  └─────────┘

  * ติดตั้ง: หันหน้าออกด้านนอกประตู
  * ระยะตรวจจับ: ปรับ ~3 เมตร
  * หน่วงเวลา: ปรับ ~3 วินาที

PIR 2 - ด้านในประตู

  HC-SR501          ESP32
  ┌─────────┐
  │  VCC ───│─────── 3.3V
  │  OUT ───│─────── GPIO 26
  │  GND ───│─────── GND
  └─────────┘

  * ติดตั้ง: หันหน้าเข้าด้านในห้อง
  * ระยะตรวจจับ: ปรับ ~3 เมตร
  * หน่วงเวลา: ปรับ ~3 วินาที

  ** สำคัญ **
  Relay ใช้ขั้ว NO (Normally Open):

  - Relay ON  → วงจรปิด  → กลอนปลดล็อก
  - ไฟดับ = Fail-Safe → กลอนปลดล็อกอัตโนมัติ

7. การติดตั้งระบบ

ขั้นตอนที่ 1: สร้างฐานข้อมูล

เปิด Laragon → Start All → เปิด phpMyAdmin

-- เปิด http://localhost/phpmyadmin
-- คลิก Import → เลือกไฟล์ database/schema.sql

-- วิธีที่ 2: Command line
mysql -u root < database/schema.sql

8. API Reference

Python Face Server (Port 5000)

GET /api/status

สถานะระบบ: ประตู, กล้อง, เซ็นเซอร์, จำนวนคน

GET /api/stream/outside

MJPEG stream กล้องด้านนอก

GET /api/stream/inside

MJPEG stream กล้องด้านใน

POST /api/door/unlock

สั่งเปิดประตู (ส่งคำสั่งไป ESP32)

POST /api/door/lock

สั่งล็อกประตู

POST /api/motion

รับข้อมูลจากเซ็นเซอร์ PIR (ESP32 → Pi)

GET /api/stats

สถิติสำหรับ Dashboard

GET /api/logs/recent?limit=50

ประวัติเข้า-ออกล่าสุด

GET /api/alerts/recent?limit=20

แจ้งเตือนล่าสุด

ESP32 HTTP Server (Port 80)

POST /api/door/unlock

สั่ง Relay ON → ปลดล็อกประตู

POST /api/door/lock

สั่ง Relay OFF → ล็อกประตู

GET /api/status

สถานะ: ประตู, PIR, WiFi RSSI, uptime

GET /ping

Health check → ตอบ "pong"

9. สิ่งที่ปรับปรุงจากต้นฉบับ

เทคนิคที่ปรับปรุงให้ดีกว่าในเอกสาร Presentation เดิม

Thread-safe Face Recognition

ต้นฉบับ: single-thread, กล้องตัวเดียว → ปรับปรุง: multi-thread, กล้อง 2 ตัวทำงานขนานกัน ไม่บล็อกกัน ใช้ threading.Lock ป้องกัน race condition

Confidence Score (% ความมั่นใจ)

ต้นฉบับ: แสดงแค่ชื่อ Known/Unknown → ปรับปรุง: คำนวณ confidence จาก face_distance แปลงเป็น % แสดงบน Dashboard

Motion-triggered Recognition

ต้นฉบับ: process ทุก 60 เฟรม (คงที่) → ปรับปรุง: ใช้ MOG2 Background Subtraction ตรวจจับ motion ก่อน จึงสั่ง Face Recognition เฉพาะเมื่อมีคนเคลื่อนไหว ประหยัด CPU มากขึ้น

HTTP REST API แทน Serial

ต้นฉบับ: แนะนำ Serial/UART → ปรับปรุง: ใช้ WiFi HTTP ทั้งหมด ทำให้ ESP32 และ Pi ไม่ต้องอยู่ติดกัน ขยายระบบง่าย รองรับหลายอุปกรณ์

Web Dashboard + MJPEG Streaming

ต้นฉบับ: ใช้ XAMPP + หน้าเว็บเบื้องต้น → ปรับปรุง: ใช้ Laragon + Tailwind CSS Dashboard สวยงาม, MJPEG stream ดูกล้องสดผ่าน browser ได้ทันที

Anomaly Detection System

ต้นฉบับ: ไม่มี → เพิ่มใหม่: ตรวจจับ Tailgating, Unknown Face, Forced Entry, Sensor Mismatch แยก severity 4 ระดับ

Snapshot System

ต้นฉบับ: ไม่มีการบันทึกภาพ → เพิ่มใหม่: บันทึกภาพถ่ายอัตโนมัติทุกครั้งที่มีการเข้า-ออก + เมื่อเกิดเหตุผิดปกติ

10. คอนเซปต์เพิ่มเติม: Dual Sensor + Dual Camera

ทำไมต้องมี 2 เซ็นเซอร์ + 2 กล้อง?

เมื่อมีบุคคลเข้า-ออกมาพร้อมกัน ให้เซ็นเซอร์เป็นตัวจับเพื่อบอกว่าใครมาก่อน-มาหลัง และให้ประตูด้านใดต้องเปิดก่อน พร้อมทั้งบันทึกภาพเพื่อตรวจสอบว่ามีการแทรกเข้า-ออกที่ผิดปกติหรือไม่

สถานการณ์ที่ระบบจัดการ

สถานการณ์ปกติ: คนเข้า 1 คน

PIR Outside ทำงาน → Camera 1 จับหน้า → ตรวจสอบสิทธิ์ → เปิดประตู → PIR Inside ทำงาน → Camera 2 บันทึกภาพขาเข้า

สถานการณ์ปกติ: คนออก 1 คน

PIR Inside ทำงาน → Camera 2 จับหน้า → บันทึกว่า "ออก" → เปิดประตู → PIR Outside → Camera 1 บันทึกภาพขาออก

สถานการณ์: เข้า-ออกพร้อมกัน

PIR ทั้ง 2 ตัวทำงานพร้อมกัน → ระบบตรวจจับทั้ง 2 ฝั่ง → ให้สิทธิ์ "คนมาก่อน" ตาม timestamp ของ PIR → เปิดประตู → บันทึกทั้ง 2 คน

สถานการณ์ผิดปกติ: Tailgating (แอบเข้าตาม)

Camera ตรวจพบ >1 คนในเฟรมเดียว แต่มีคนไม่รู้จัก → แจ้งเตือน TAILGATING severity CRITICAL → บันทึก Snapshot → แจ้งผู้ดูแลผ่าน Dashboard

สถานการณ์ผิดปกติ: ใบหน้าไม่รู้จัก

Camera จับภาพได้แต่ไม่ตรงกับฐานข้อมูล → แจ้งเตือน UNKNOWN_FACE → บันทึก Snapshot → ไม่เปิดประตู → แสดงสถานะ "ปฏิเสธ"
